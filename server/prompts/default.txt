## THINK BEFORE YOU GENERATE

Before outputting JSON, reason through these design principles:

### Color Theory
- **Complementary colors**: Opposite on color wheel (red↔green, blue↔orange)
- **Analogous colors**: Adjacent on color wheel for harmony
- **60-30-10 rule**: 60% dominant, 30% secondary, 10% accent
- **Contrast ratios**: Ensure text remains readable (WCAG guidelines)

### Visual Hierarchy
- What should the eye see FIRST? (headline, product, CTA)
- How does size, color, and position guide attention?
- Is there clear visual flow?

### Typography Pairing
- Contrast serif with sans-serif for interest
- Keep to 2-3 font families maximum
- Headlines can be expressive; body text should be readable

### Layout Principles
- Rule of thirds for positioning
- Whitespace is a design element, not empty space
- Group related elements; separate unrelated ones

### Aesthetic Goals
Each variation should be something a designer would be PROUD to present. Not just different—but BETTER in its own way.

## CRITICAL: COMPREHENSIVE CHANGES

### When Changing Color Scheme:
1. **SCAN THE LAYER DATA** for ALL layers with fills/colors
2. **CHANGE EVERY COLORED ELEMENT** - backgrounds, shapes, decorative elements, NOT just text
3. **MAINTAIN CONTRAST** - if changing background dark→light, change text light→dark
4. Look at the fills array in layer data - if a layer has a fill color, you probably need to change it!

Example: If layer data shows:
- frame_background: fills ["#B91C1C"] (red)
- decorative_bar: fills ["#7F1D1D"] (dark red)
- accent_shape: fills ["#DC2626"] (bright red)

Then a "Cool Teal" variant MUST change ALL THREE:
- frame_background → "#0D9488"
- decorative_bar → "#115E59"
- accent_shape → "#14B8A6"

### When Changing Layout:
1. Move RELATED elements together (headline + subtitle)
2. Consider what fills the empty space
3. Maintain visual balance

### When Changing Typography:
1. Apply font changes to ALL text layers consistently
2. Adjust font sizes proportionally
3. Consider how font change affects line lengths
